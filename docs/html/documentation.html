Idea Generator - Documentazione

Benvenuto nella documentazione di Idea Generator, un sistema automatizzato per la generazione e pubblicazione di idee per applicazioni software.

Funzionalità Principali

  • Generazione automatica di idee per applicazioni
  • Pubblicazione automatica su GitHub
  • Integrazione con AutoDev per lo sviluppo
  • Pianificazione giornaliera delle pubblicazioni
  • Logging dettagliato delle operazioni

Requisiti di Sistema

  • Node.js v18 o superiore
  • npm v8 o superiore
  • Accesso a GitHub con token API
  • AutoDev configurato (opzionale)

Configurazione

Installazione Dependencies

npm install

Configurazione GitHub Token

Creare un file .env nella root del progetto:

GITHUB_TOKEN=il_tuo_token_qui

Generazione Idee

Per generare idee manualmente:

node scripts/generateIdea.js
Le idee vengono generate in base alle categorie predefinite nel sistema.

Pubblicazione su GitHub

Le idee vengono automaticamente pubblicate come issue nel repository configurato.

Ogni idea include:

  • Titolo e descrizione
  • Obiettivo principale
  • Funzionalità chiave
  • Requisiti tecnici
  • Timeline di sviluppo
  • Risorse necessarie

Pianificazione Automatica

Il sistema può essere configurato per generare e pubblicare idee automaticamente:

node scripts/dailyGenerator.js
Per impostazione predefinita, le idee vengono generate alle 9:00 AM ogni giorno.

Integrazione con GitHub

Il sistema utilizza l'API di GitHub per:

  • Creare nuove issue per le idee
  • Gestire le etichette delle issue
  • Tracciare lo stato delle idee

Integrazione con AutoDev

Ogni idea generata include un prompt ottimizzato per AutoDev, contenente:

  • Requisiti tecnici dettagliati
  • Stack tecnologico consigliato
  • Timeline di sviluppo
  • Risorse necessarie

Gestione Logs

I logs vengono salvati nella directory logs/ e includono:

  • Log di generazione idee
  • Log di pubblicazione
  • Log di errore

Risoluzione Problemi

Errori Comuni

  • Errore di autenticazione GitHub: Verificare il token nel file .env
  • Errore di generazione: Controllare i logs per dettagli
  • Errore di pubblicazione: Verificare i permessi del repository
