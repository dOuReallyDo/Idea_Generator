Integrazione Idea Generator e AutoDev

Guida completa alla configurazione e utilizzo integrato dei sistemi

Panoramica dell'Integrazione

Il sistema di integrazione tra Idea Generator e AutoDev permette la generazione automatica di idee per applicazioni e il loro sviluppo tramite agenti AI senza intervento umano. Il flusso di lavoro è completamente automatizzato.

Questa integrazione crea un ciclo completo di innovazione: dall'idea alla pubblicazione del codice, tutto gestito automaticamente.

Architettura del Sistema

Componenti Principali

  • Idea Generator: Genera idee innovative per applicazioni basate su diverse categorie
  • AutoDev: Sistema di sviluppo automatico che implementa le idee generate
  • GitHub Integration: Middleware che collega i due sistemi tramite repository GitHub

Flusso dei Dati

  1. Idea Generator crea nuove idee e le pubblica come issue su GitHub
  2. Le issue vengono etichettate come "non-sviluppato"
  3. AutoDev rileva le issue con questa etichetta e avvia lo sviluppo
  4. Durante lo sviluppo, l'etichetta cambia in "in-sviluppo"
  5. Al completamento, l'etichetta diventa "sviluppo-completato"

Configurazione dell'Integrazione

Prerequisiti

  • Node.js v18 o superiore
  • Account GitHub con permessi per creare repository
  • Token GitHub con permessi "repo" completi
  • Accesso a un LLM (locale tramite Ollama o cloud tramite API)

Configurazione dei File .env

Per Idea Generator (~/Idea_Generator/.env):

GITHUB_TOKEN=il_tuo_token_qui
GITHUB_USERNAME=dOuReallyDo
IDEA_REPO=daily-app-ideas

Per AutoDev (~/AutoDev/.env):

GITHUB_USERNAME=dOuReallyDo
GITHUB_TOKEN=il_tuo_token_qui
PREFERRED_IDE=cursor
WORKSPACE_ROOT=/Users/mariocurcio/AutoDev/workspace
MAX_CONCURRENT_PROJECTS=3
DEVELOPMENT_TIME_PER_PROJECT=7200000
IDEA_GENERATOR_REPO=daily-app-ideas
ANTHROPIC_API_KEY=la_tua_chiave_api_qui

Esecuzione Automatica

Il sistema è configurato per l'esecuzione automatica giornaliera tramite crontab:

# Generazione automatica di idee ogni giorno alle 9:00
0 9 * * * cd /Users/mariocurcio/Idea_Generator && /usr/local/bin/node scripts/dailyGenerator.js >> /Users/mariocurcio/Idea_Generator/logs/cron.log 2>&1

# Esecuzione AutoDev ogni giorno alle 10:00
0 10 * * * cd /Users/mariocurcio/AutoDev && /usr/local/bin/node src/index.js >> /Users/mariocurcio/AutoDev/logs/cron.log 2>&1

Questo significa che:

  1. Ogni giorno alle 9:00, il sistema genererà 5 nuove idee e le pubblicherà su GitHub
  2. Alle 10:00, AutoDev verificherà se ci sono nuove idee non sviluppate e inizierà a lavorare su di esse
Assicurarsi che il computer sia acceso e connesso a Internet negli orari programmati per l'esecuzione, altrimenti il sistema non potrà funzionare correttamente.

Monitoraggio e Manutenzione

File di Log

I file di log permettono di monitorare l'attività del sistema:

  • ~/Idea_Generator/logs/combined.log: Log completo di Idea Generator
  • ~/Idea_Generator/logs/error.log: Errori di Idea Generator
  • ~/Idea_Generator/logs/cron.log: Output dell'esecuzione programmata
  • ~/AutoDev/logs/combined.log: Log completo di AutoDev
  • ~/AutoDev/logs/error.log: Errori di AutoDev
  • ~/AutoDev/logs/cron.log: Output dell'esecuzione programmata

Manutenzione Regolare

Procedure consigliate per la manutenzione del sistema:

  1. Verificare regolarmente lo spazio disponibile in ~/AutoDev/workspace
  2. Controllare la validità del token GitHub (scade dopo un certo periodo)
  3. Monitorare il saldo delle API cloud se si utilizzano LLM cloud
  4. Effettuare backup regolari dei progetti completati

Risoluzione Problemi

Problemi Comuni

Problema Possibile Causa Soluzione
Errore "Bad credentials" in Idea Generator Token GitHub non valido o scaduto Generare un nuovo token e aggiornare il file .env
AutoDev non rileva nuove idee Etichetta errata o permessi repo insufficienti Verificare le etichette delle issue e i permessi del token
Problemi con il crontab Path errato o permessi insufficienti Verificare i path nel crontab e i permessi di esecuzione
Errori con LLM in AutoDev API key scaduta o servizio non disponibile Verificare lo stato dell'API e aggiornare le chiavi
